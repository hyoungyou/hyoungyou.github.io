QCP-T237 : 수강 기간 필터 검증

시나리오: 다중 날짜 기준(시작일/종료일/기간 내 등) 필터링 결과 대조

실제 결과: PASS

⚠️ 수행 지연 사유:
1. 검증 포인트 과다 : 단일 TC 내 검증 포인트 4개 존재 (세부 날짜 필터링 결과 대조). 개별 스텝 수행 및 데이터 확인에 물리적 시간 집중됨
2. 기획 모호성 : "필터 조건(시작일/종료일/기간 내 등)"에 대한 정의에 대한 설명이 없어 이해 하는데 어려움 발생

QCP-T223 : 수강 상태 필터 검증

시나리오: 6가지 수강 상태 필터링 확인

실제 결과: PASS (단, 테스트 과정 중 MongoDB와 SQLDB 간 동기화 지연으로 인한 데이터 누락 현상 확인 및 재검증 완료)

⚠️ 수행 지연 사유:
1. 검증 포인트 과다 : 단일 TC 내 검증 포인트 6개 존재 (상태별 필터링 결과 대조). 개별 스텝 수행 및 데이터 확인에 물리적 시간 집중됨.
2. 인프라 이슈 : 테스트 중 MongoDB-SQLDB 간 동기화 지연으로 인한 데이터 누락 현상이 발견되어, 동기화 대기 및 재현 테스트를 반복함에 따라 수행 시간이 크게 증가함.

QCP-T6192 : 수료증 이미지 검증

시나리오: 수료 기준 충족 시 수료증 버튼 노출 및 수료증 이미지 출력 확인

실제 결과: FAIL (직인 이미지 깨짐 발생)

⚠️ 수행 지연 사유:
1. 준비 지연 : 수료증 버튼이 노출되기 위한 선행 조건(수강률 100%, 시험 통과 등)을 만족시키기 위한 테스트 데이터 빌드 및 상태값 변경 작업에 시간 소요.
2. 결함 분석 : 수료증 직인 이미지가 깨져 출력되는 결함이 발견되어, 원인 파악을 위한 추가적인 테스트 과정이 포함됨.
↑ 요약표로 돌아가기

QCP-T2943 : 수료증 번호 형식 변경에 따른 계정별 수료 번호 적용 정합성 검증

시나리오: 수료 번호 형식 변경 시 기존/신규 발급자 데이터 영향도 평가

실제 결과: FAIL (기발급 수료증의 수료 번호가 임의의 값으로 출력)

⚠️ 수행 지연 사유:
1. 복합 시나리오 : 수료 번호 업데이트 정책(기존 유지 vs 전체 변경)에 대한 복합 시나리오 검증을 위해 다수의 테스트 계정 생성 및 데이터 정합성 확인이 필요했음.
2. 결함 분석 : 기발급 수료증의 수료번호가 임의의 값으로 출력되는 현상이 발견되어, 원인 파악을 위한 추가적인 테스트 과정이 포함됨.
↑ 요약표로 돌아가기

QCP-T3629 : 카테고리 삭제 시 안내 다이얼로그 노출 검증

시나리오: 상품태그 연결된 카테고리 삭제 시 안내 다이얼로그 노출 여부 확인

실제 결과: N/T (삭제된 기능, TC 최신화 안됨)

⚠️ 수행 지연 사유:
1. 결함 분석 : 안내 다이얼로그 없이 카테고리가 삭제되어버리는 현상이 발견 되어, 원인 파악을 위한 추가적인 테스트 과정이 포함됨.
2. TC 오류 : 이전 버전에서는 존재하던 다이얼로그가 최신 버전에서 미노출 되는 것인지, TC 이해도가 부족한 것인지 파악하는데 추가적인 테스트 과정이 포함됨.
